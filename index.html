Number Format

The number format in chmod codes represents the combination of permissions assigned to the user, group, and others.

Each digit in the three-digit sequence corresponds to a specific set of permissions:

  • First Digit (User): Represents the permissions for the file owner.
  • Second Digit (Group): Represents the permissions for the group associated with the file.
  • Third Digit (Others): Represents the permissions for everyone else.

The digits follow a numeric value assignment:

  • 4 (Read): Permission to read the file or directory.
  • 2 (Write): Permission to modify or delete the file or directory.
  • 1 (Execute): Permission to execute the file or access the directory.

Combining these values results in a numeric chmod code. For example, a code of 777 means full read, write and execute permissions for the user, group, and others.

Text Format

In the text format, chmod codes use a combination of letters to represent permissions:

  • 'r' (Read): Denoted by 'r', it allows the user to read the content of a file or view the contents of a directory.
  • 'w' (Write): Denoted by 'w', the user can modify or delete the file or add/remove the content in a directory.
  • 'x' (Execute): Denoted by 'x', it allows the user to execute a file or access the contents of a directory.

The text format consists of three sets of three characters each, representing the user, group, and others. For example, 'rwxrwxrwx' signifies that the user, group, and others have read, write, and execute permissions.
